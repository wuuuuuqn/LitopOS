LitopOS Documentation

MicroPython Embedded Shell for ESP32 -- 354 commands, drive system, auth, app manager, web server.

Overview

LitopOS is a micro operating system running on ESP32 with MicroPython. It provides a Unix-like shell experience with 354 commands, an A-Z drive letter system, user authentication, a third-party app manager, and a built-in AP web server.

Requirements ESP32 / ESP32-S2 / ESP32-S3 / ESP32-C3 with MicroPython firmware (1.19+).

Installation

Flash MicroPython Firmware

Download ESP32 MicroPython firmware and flash it with esptool.

Upload LitopOS Files

Upload the project to the ESP32 flash filesystem using ampy or Thonny.

Reset

Press the RST button. LitopOS boots automatically from main.py.

bash 
# Install esptool
pip install esptool adafruit-ampy

# Erase flash and flash MicroPython firmware
esptool.py --chip esp32 erase_flash
esptool.py --chip esp32 write_flash -z 0x1000 micropython-firmware.bin

# Upload LitopOS (18 files)
ampy --port /dev/ttyUSB0 put boot.py
ampy --port /dev/ttyUSB0 put main.py
ampy --port /dev/ttyUSB0 mkdir litopos

shell

sys_info          # System information

whoami            # Current user
app_list          # List installed apps

Drive System

On first boot, LitopOS scans all flash partitions and mounts filesystem partitions (SPIFFS, LittleFS, FAT) as drive letters A through Z. Drive A: is the default -- all files are saved there unless you specify otherwise.

Default Drive A: is always the default drive. It contains .app/ for third-party apps and .auth.json for user data.

Directory Structure

text
A:/
  .app/            Third-party applications
  .auth.json       User database
  config.json      System configuration
  startup.py       Auto-run script

Authentication

LitopOS includes a user authentication system with role-based access control. System files (/boot.py, /main.py, /litopos/) are protected and can only be deleted by admin users.

Default Account

Security Note Default credentials are admin / admin. Change the password immediately with passwd admin newpass.

Commands

App Manager

Third-party Python applications are stored in A:/.app/. Apps are simple .py files that can be installed, listed, and run with dedicated commands.

Commands

CommandDescription
app_install <file.py> [name]Install a .py file into .app/
app_uninstall <name>Remove an app (admin only)
app_list / appsList all installed apps
app_info <name>Show app details
app_run <name> / run <name>Execute an installed app
shell 
# Install an app
LitopOS> app_install sensor_reader.py
Installed 'sensor_reader' -> A:/.app/sensor_reader.py (512 bytes)

# Run it
LitopOS> run sensor_reader
App 'sensor_reader' finished

# List all apps
LitopOS> apps
Installed Apps (1):
  sensor_reader    512B  (run 'sensor_reader')

Web Server

LitopOS includes a built-in HTTP server that runs in AP (Access Point) mode. When started, it creates a WiFi hotspot and serves HTML pages to connected clients.

Commands

CommandDescription
web_start [port]Start AP + web server (default port 80)
web_stopStop the web server
web_statusShow server status, IP, and route count
web_page <file.html> [path]Set a custom HTML page from file
web_file <file> [path]Serve a file at a URL path
web_dir <dir> [base]Serve all files from a directory
web_route <path> <html>Add a custom URL route
web_clearRemove all custom routes
shell 
# Start web server on port 80
LitopOS> web_start 80
AP Web Server started!
  SSID: 'LitopOS' | IP: 192.168.4.1 | Port: 80
  Open http://192.168.4.1/ in browser

# Set custom page
LitopOS> web_page dashboard.html
Page set for / (2048 bytes)

HTTP API

When the web server is running, these REST endpoints are available:

EndpointMethodResponse
/GETHTML page (default or custom)
/api/sysinfoGETJSON: chip, freq, mem_free, uptime
/api/memGETJSON: free, alloc, total memory
/api/gpioGETJSON: all GPIO pin states
/api/commandsGETJSON: command list and count
bash 
curl http://192.168.4.1/api/sysinfo
{"chip":"ESP32","freq":240,"mem_free":102400,"uptime":3600}

curl http://192.168.4.1/api/mem
{"free":102400,"alloc":51200,"total":153600}
